			- 1. 数据库层面的多租户：
				- 独立数据库： 为每个租户分配一个独立的数据库实例。这种方法隔离性最好，但成本也最高。

			- 講一下拿來做什麼, SSO? 如何實現
				- 重點是在SSO服務器上檢查session認證是否有效
				- 用户访问应用 A： 用户尝试访问应用 A，但未登录。
				- 重定向到 SSO 服务器： 应用 A 将用户重定向到 SSO 服务器。
				- 用户认证： 用户在 SSO 服务器上进行身份验证（通常是输入用户名和密码）。
				- SSO 服务器颁发授权码： SSO 服务器验证用户身份后，向应用 A 颁发授权码（Authorization Code）。
				- 应用 A 获取访问令牌： 应用 A 使用授权码向 SSO 服务器交换访问令牌（Access Token）。
				- 应用 A 访问用户信息： 应用 A 使用访问令牌从 SSO 服务器获取用户信息。
				- 用户访问应用 B： 用户尝试访问应用 B，同样未登录。
				- 重定向到 SSO 服务器： 应用 B 将用户重定向到 SSO 服务器。
				- SSO 服务器验证用户会话： SSO 服务器检查用户的会话是否有效（例如，通过 Session Cookie）。
				- SSO 服务器颁发访问令牌： 如果会话有效，SSO 服务器直接向应用 B 颁发访问令牌。
				- 应用 B 访问用户信息： 应用 B 使用访问令牌从 SSO 服务器获取用户信息。

- 可以講講什麼是GitOPS
	- 工作流程
		- GitOps：
			- 声明式配置： 开发人员将系统的期望状态（例如应用版本、基础设施配置）以声明式的方式描述在 Git 仓库中。
			- Git 变更触发： 当 Git 仓库中的配置发生变更时，GitOps operator（例如 Argo CD 或 Flux）会自动检测到这些变更。
			- 自动同步： Operator 会将系统的实际状态与 Git 仓库中声明的期望状态进行比较，并自动进行协调，确保实际状态与期望状态一致。
		- 传统 CI 自动化：
			- 代码提交触发： 开发人员将代码提交到版本控制系统（如 Git），触发 CI 流程。
			- 流水线执行： CI 服务器（如 Jenkins）根据预定义的流水线执行构建、测试和部署等任务。
			- 人工干预： 部署到生产环境通常需要人工审批或手动触发。
	- 具体实践
		- GitOps：
			- Pull-based 部署： GitOps operator 主动从 Git 仓库拉取配置变更，而不是像传统 CI 那样将变更推送到环境中。
			- 不可变基础设施： 鼓励使用不可变基础设施，即通过销毁和重建来更新基础设施，而不是直接修改现有基础设施。
			- 声明式工具： 使用 Kubernetes 原生的声明式工具（如 Helm、Kustomize）来描述和管理应用。
	- 传统 CI 自动化：
		- Push-based 部署： CI 服务器将构建好的制品推送到目标环境中。
		- 可变基础设施： 允许直接修改现有基础设施。
		- 脚本化工具： 通常使用脚本或命令行工具来执行部署任务。

**服务发现与注册：**

- **Netflix Eureka:**
    - 替代方案：
        - **Spring Cloud Consul:** 成熟的服务发现和配置管理工具，与 Spring Cloud 生态系统集成良好。
        - **Spring Cloud Zookeeper:** 另一个流行的服务发现和协调工具，提供 Spring Cloud 集成。
        - **Hashicorp Consul:** 服务网格解决方案，提供服务发现、配置管理和分段功能。
        - **Nacos:** 阿里巴巴开源的服务发现、配置管理和服务管理平台。

**客户端负载均衡：**

- **Netflix Ribbon:**
    - 替代方案：
        - **Spring Cloud LoadBalancer:** Spring Cloud 官方提供的客户端负载均衡解决方案。
        - **Resilience4j:** 轻量级容错库，提供断路器、重试、限流等功能，可与 Spring Cloud LoadBalancer 结合使用。

**声明式 REST 客户端：**

- **Netflix Feign:**
    - 替代方案：
        - **Spring Cloud OpenFeign:** Feign 的开源替代品，与 Spring Cloud 生态系统集成良好。

**API 网关：**

- **Netflix Zuul:**
    - 替代方案：
        - **Spring Cloud Gateway:** Spring Cloud 官方提供的 API 网关解决方案。
        - **Apache APISIX:** 高性能、可扩展的 API 网关，支持动态路由、插件化等功能。

**断路器：**

- **Netflix Hystrix:**
    - 替代方案：
        - **Resilience4j:** 提供断路器、重试、限流等功能。
        - **Sentinel:** 阿里巴巴开源的流量控制和熔断框架。

**分布式配置管理：**

- **Spring Cloud Config:** （虽然属于Spring Cloud项目，但通常与Netflix组件一起使用）
    - 替代方案：
        - **Apollo:** 携程开源的配置管理平台，功能强大且易于使用。
        - **Nacos:** 阿里巴巴开源的服务发现、配置管理和服务管理平台。

**链路追踪：**

- **Spring Cloud Sleuth + Zipkin:** （Zipkin 虽然不是 Netflix 项目，但通常与 Spring Cloud Sleuth 结合使用）
    - 替代方案：
        - **Spring Cloud Sleuth + Jaeger:** Jaeger 是 Uber 开源的分布式追踪系统，提供更丰富的功能和可视化界面。
